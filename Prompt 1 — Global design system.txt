Prompt 1 — Global design system

Copy this first.

Create a complete mobile-first design system for a Bangla-first tuberculosis field workflow app called “TB-FO Assistant”.

The app is for BRAC-style TB Field Organisers who manage TB patients, DOT tracking, lab results, contact investigation, TPT, treatment outcome, reports, and automatic FO daily diary.

Design style:
- Professional public-health field app
- Clean, trustworthy, calm, medical, government/NGO-friendly
- Inspired by BRAC green and community health systems
- Mobile-first, responsive for 360px, 390px, 430px, tablet, and desktop
- Pixel-polished, high contrast, easy to use outdoors
- Bangla labels with English medical/programme terms where needed
- Rounded cards, clear hierarchy, soft shadows, excellent spacing
- Use icons lightly: patient, lung, calendar, lab, diary, report, warning
- Avoid clutter, but support dense field data

Color palette:
- Primary green: #0A6E4F
- Dark green: #074F39
- Accent green: #1A8A64
- Light green background: #E8F5F0
- Blue for lab/DOT: #1A4F7A
- Red for urgent/overdue: #B83232
- Amber for due soon: #B5760A
- Purple for contact investigation/TPT: #6B3FA0
- Background: #EEF4F1
- Card: #FFFFFF
- Text: #1C2B26
- Muted text: #5A7268
- Border: #CCDDD6

Typography:
- Use Hind Siliguri for Bangla UI.
- Use a monospace style for TR numbers and numeric badges.
- Font sizes must be mobile-friendly.
- Use strong labels, readable inputs, and clear button states.

Components to design:
1. Sticky mobile top app bar
2. Bottom mobile navigation bar
3. Desktop sidebar/navigation variant
4. Summary metric cards
5. Patient cards
6. Search and filter controls
7. Collapsible form section cards
8. Input fields, select fields, textareas
9. Radio button chips
10. Status badges
11. DOT calendar grid cells
12. Timeline/diary items
13. Alert cards
14. Report cards
15. Primary/secondary/danger/download buttons
16. Empty states
17. Toast notification
18. Modal confirmation
19. Floating action button for new patient on mobile

Responsive behavior:
- On mobile, use a bottom nav with 5 items: Dashboard, Patients, Today, Diary, Reports.
- On desktop, use a left sidebar and wider content area.
- Forms should be single-column on mobile, two-column on tablet, three/four-column on desktop.
- Tables should become cards on mobile.
- Large DOT grid should support horizontal scroll on mobile but remain visually clean.

Create design tokens, component styles, and a sample preview screen showing these components together.
